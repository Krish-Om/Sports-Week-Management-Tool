4. Prisma Schema (The Source of Truth)
Code snippet
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

generator client {
  provider = "prisma-client-js"
}

enum Role {
  ADMIN
  MANAGER
}

enum MatchStatus {
  UPCOMING
  LIVE
  FINISHED
}

model Profile {
  id              String   @id @default(uuid())
  fullName        String?
  role            Role     @default(MANAGER)
  assignedGameId  String?
  assignedGame    Game?    @relation(fields: [assignedGameId], references: [id])
}

model Game {
  id       String    @id @default(uuid())
  name     String    @unique // e.g., Football, Valorant
  icon     String?   // Lucide icon name
  profiles Profile[]
  matches  Match[]
}

model Team {
  id           String  @id @default(uuid())
  name         String
  department   String?
  matchesAsA   Match[] @relation("TeamA")
  matchesAsB   Match[] @relation("TeamB")
}

model Match {
  id          String      @id @default(uuid())
  gameId      String
  game        Game        @relation(fields: [gameId], references: [id])
  teamAId     String
  teamA       Team        @relation("TeamA", fields: [teamAId], references: [id])
  teamBId     String
  teamB       Team        @relation("TeamB", fields: [teamBId], references: [id])
  venue       String
  startTime   DateTime
  status      MatchStatus @default(UPCOMING)
  scoreA      Int         @default(0)
  scoreB      Int         @default(0)
  winnerId    String?
}
5. 5-Day Execution Plan
Day 1: Scaffolding & Database
Initialize Next.js with npx create-next-app@latest.

Install Prisma: npm install prisma @prisma/client.

Connect Supabase via .env.

Push schema: npx prisma db push.

Setup Shadcn UI components (Card, Button, Badge, Table).

Day 2: Auth & Role Logic
Setup Supabase Auth.

Create a middleware.ts to protect /manager and /admin routes.

Build the "Admin Only" view to create Teams and Games and assign Managers.

Day 3: Manager Dashboard (The Field App)
Build the Manager view: Filter matches by assignedGameId.

Create the "Create Match" form (Step 2 of the plan).

Implement the "Live Score Toggle" so managers can update scores from their phones.

Day 4: Public Dashboard (The "Vibe" Front)
Build the root page / showing all fixtures.

Implement Supabase Realtime so the page refreshes scores automatically without a reload.

Add a "Live" pulse animation for matches with status: LIVE.

Day 5: Polish & Deployment
Add search and category filters.

Mobile responsiveness check (Managers will be on the field).

Deploy to Vercel.
